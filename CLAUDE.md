# このプロジェクトの開発ワークフロー

## 役割分担（重要）

大きな機能実装は **Claude が Codex CLI（`codex exec`）を直接実行**して任せる。
Claude 自身の役割：

- **設計・タスク分解**: 要望をタスクに分解し、Codex への指示（プロンプトまたは `docs/` の指示書）を書いて `codex exec` で実行する
- **コードレビュー**: Codex の成果物のレビュー（バグ・設計・パフォーマンス）
- **実機検証**: 実際にアプリを動かして動作確認する（下記「検証方法」参照）
- **軽微〜中程度の修正は直接コミットしてよい**
- **画像アセット（GPT Image 2 / image 2.0）**: Higgsfield ではなく Codex 経由で生成する（クレジット節約のため）

## プロジェクト概要

お金を賭けないコイン制の競馬ベットゲーム（日本語UI）。ビルド工程なしの静的サイト。

- **技術**: Vanilla JS (ES Modules) + three.js（importmap で unpkg から読込）+ Firebase Firestore（オンライン対戦のみ）
- **起動**: 任意の静的サーバーで配信するだけ（例: `python3 -m http.server`）。ビルド不要
- **モード**: ローカル対戦（1台を回す）とオンライン対戦（合言葉ルーム、Firebase 設定が必要）

### 主要ファイル

| ファイル | 役割 |
|---|---|
| `src/engine.js` | レース生成・馬券精算のコアロジック |
| `src/race.js` | レース事前計算シミュレーションと再生プレイヤー（2Dフォールバック込み） |
| `src/race3d.js` | three.js による3Dレース描画（動的カメラ・コース装飾・紙吹雪など） |
| `src/betui.js` / `src/raceui.js` | ベット画面・レース/結果画面のUI |
| `src/local.js` / `src/online.js` | ローカル/オンラインの進行管理 |
| `src/horses.js` / `src/bets.js` | 馬データ・賭け式定義 |
| `style.css` | 全画面のスタイル（ゴールド基調のリッチテーマ） |

### 検証方法（ヘッドレス環境での注意）

- Playwright + 同梱 Chromium で実プレイ検証ができる（ローカル対戦: ホーム → ローカル → 馬選択 → ベット → レース → 結果）
- リモート実行環境では CDN（unpkg / threejs.org）に Chromium から直接届かないことがある。
  その場合は `node_modules/three` をローカルに置き、Playwright の `page.route()` で
  unpkg URL をローカルファイルに差し替える。馬の GLB モデルは失敗しても
  フォールバック描画になるため検証は可能
- レース中のカメラは全馬（1位〜最下位）が常に画面内に収まる動的カメラ。
  カメラ関連を変更したら、馬がフレームアウトしないことを必ず確認すること
