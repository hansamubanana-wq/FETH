# Codex実装指示書: スマホ横画面対応＋モバイル最高品質化（v0.11.0）

## 1. スマホは横画面前提にする（優先度高）

現状スマホ縦だとレース画面が狭くて見づらい。**横画面（landscape）を標準の遊び方**にする。

- `manifest.webmanifest` の `orientation` を `landscape` に変更
- **縦持ち検出時のガイド**: スマホ（coarse pointer かつ短辺が小さい端末）で portrait のとき、
  「端末を横向きにしてください」のフルスクリーンオーバーレイ（回転アイコンのアニメーション付き、
  デザインは既存のダーク×ゴールドに合わせる）を表示。横向きになったら自動で消える。
  レース画面だけでなくアプリ全体で出してよいが、**ホーム画面と結果画面は縦でも閲覧できて構わない**
  （最低限レース画面と馬選択画面では横を促す）
- 横画面時のレイアウト最適化:
  - レース画面: 3Dキャンバスを画面いっぱいに使い、HUD（順位表・ログ・速報）は左右端に
    オーバーレイ配置してキャンバスを潰さない。セーフエリア（ノッチ）対応 `env(safe-area-inset-*)`
  - 馬選択/ベット画面: 横幅を活かして馬カードを4列、賭け式・金額パネルは右側または下部に
  - 100svh / 100dvh を使い、iOS Safariのアドレスバーで見切れないこと

## 2. モバイルでもグラフィックは最高品質（優先度高）

現在 `src/race3d.js` で `isMobile` による品質ダウングレードが多数入っている（該当箇所を洗い出して見直す）:

- `setPixelRatio` の上限（現状モバイル1.5）→ デスクトップと同じ 2 に
- `shadowMap.enabled = !this.isMobile` → モバイルでも有効に
- シャドウマップ解像度 512 → 2048（負荷が厳しければ1024だが、まず2048で試す）
- 観客数・星の数・紙吹雪・砂埃パーティクル・フラッシュライトの間引き → デスクトップと同数に
- スポットライト強度などの差も撤廃

**ただし実測でフレームレートが落ちすぎないこと**が条件。以下の安全策を入れる:
- 実行時にFPSを計測し、一定時間（例: 3秒平均）30fpsを下回ったら段階的に品質を落とす
  **動的フォールバック**を実装（pixelRatio → 影 → パーティクル数 の順で削る）
- つまり「デフォルト最高品質、危なくなったら自動で落とす」設計にする。
  端末カテゴリだけで最初から落とすのはやめる

## 制約

- ビルドなし静的サイト維持、Vanilla JS + three.js (importmap)
- 動的カメラ「全馬が常に画面内」のロジックに手を入れない（横画面ではアスペクト比が変わるため、
  カメラのfitロジックが横長でも全馬を収められるか確認だけはすること）
- 2Dフォールバックを壊さない
- 完了時 `src/version.js` を APP_VERSION 0.11.0 / APP_BUILD 14 に、index.htmlのバッジも更新
- タスクごとに日本語メッセージで独立コミット。push・デプロイはしない
