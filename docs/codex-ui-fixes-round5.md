# Codex実装指示書: UI改善 第5弾（v0.8.0）

ユーザーからのフィードバック3点を修正する。スクリーンショットに基づく指摘なので、実際に画面を出して確認しながら調整すること。

## 1. 結果画面: WINNERカードと表彰台の重なり解消（優先度高）

現状: 勝利キーアート（victory.png のWINNERカード）の直下に表彰台の1位カードが食い込み、
キーアートの下端と1位カードが中途半端に重なって見える。

修正方針（例。より良い案があればそちらで良い）:
- WINNERカード（キーアート＋優勝馬肖像）と表彰台を**重ねずに縦に分離**する。
  キーアートは上部の独立したヒーローバナーとして完結させ、その下に余白を取って表彰台（2位/1位/3位）を置く
- または表彰台をやめ、キーアート内に1位を統合し、2位以下はリスト表示に一本化
- どの画面幅（375px〜ワイド）でも重なり・はみ出しがないこと

## 2. ベット画面の操作をわかりやすく（優先度高）

現状の問題: 賭け式選択 → 馬カードタップ → 金額 → 確定 の流れが画面上で伝わらず、
何をどの順に押せばいいかわかりにくい。

修正方針:
- **ステップ表示**を導入: 「① 賭け式を選ぶ → ② 馬を選ぶ → ③ 金額を決めて賭ける」を画面上部に常時表示し、現在のステップをハイライト
- 賭け式ごとに「あと何頭選ぶか」を明示（例: 馬連なら「2頭選んでください（あと1頭）」）。選択済み馬には順序バッジ
- 金額入力と「この内容で賭ける」ボタンを**ベットスリップ（賭け内容のまとめカード）**として画面下部に固定表示。
  スリップには選んだ賭け式・馬・想定払戻がリアルタイム表示され、確定ボタンはスリップ内に1つだけ
- 未完了時は確定ボタンをdisabledにし、理由をボタン内に表示（「馬をあと1頭選んでください」）
- 賭け終わったベットの一覧と「ベットを終了する」も混同しないよう視覚的に分離

## 3. レース中の大型ビジョンの位置（優先度高）

現状: コース内側の大型ビジョンが走行レーンと画面上被って、馬が見えにくい。

修正方針:
- ビジョンをコース内側の**中央奥（スタート/ゴールから遠い側）**に移動し、高さを下げるかサイズ縮小
- 動的カメラの视界（全馬が映る範囲）に入りにくい配置・角度を実際にレースを流して確認する
- それでも被るケースがあるなら、ビジョンを薄い半透明にするかHUD側に役割を移す

## 制約（毎回同じ・必読）

- ビルドなし静的サイト維持、Vanilla JS + three.js (importmap)
- レスポンシブ必須（375px〜ワイド）
- 動的カメラ「全馬が常に画面内」のロジックに手を入れない
- 2Dフォールバックを壊さない
- 完了時 `src/version.js` を APP_VERSION 0.8.0 / APP_BUILD 11 に、index.htmlのバッジも更新
- タスクごとに日本語メッセージで独立コミット。push・デプロイはしない
