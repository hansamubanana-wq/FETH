# Codex実装指示書: グラフィック強化 第2弾（v0.5.0）

## ユーザーからの指摘（最優先で直す）

1. **ホームの背景画像が小さい**: 画像が画面全体を覆っておらず、周囲が画像でない領域になっている。
   ブラウザの実ウィンドウサイズ（スマホ縦〜32インチワイド）どれでも、**ビューポート全体（100vw×100svh）を背景画像が完全に覆う**こと。
   `#app` の max-width や負マージン頼みの現行実装をやめ、`body` 直下の固定フルスクリーンレイヤー（position:fixed; inset:0）に敷く方式に変える。
2. **背景画像がホームだけ**: 全画面（プレイヤー設定・馬選択/ベット・レース・結果・オンライン各画面）に背景画像の雰囲気を行き渡らせる。

## 実装内容

### 1. 背景システムの作り直し
- GPT Image 2 で背景を追加生成し `assets/art/` へ:
  - `bg-home.png`（既存home-hero.pngの置き換え可。夜の競馬場全景、16:9・2048px級）
  - `bg-paddock.png`（パドック/厩舎の雰囲気 — 馬選択・ベット画面用）
  - `bg-stadium.png`（スタンド内観・照明 — 結果画面用）
  - 縦長スマホで上下が切れても破綻しない構図（中心にモチーフ、端は暗め）
- `body` 固定レイヤーに画面ごとの背景をクロスフェード切替（activeスクリーンに応じてクラス切替、opacity transition 0.4s）
- どの画面でも読みやすさ優先: 背景の上に暗色オーバーレイ＋パネルは半透明ダーク＋blur

### 2. さらなるグラフィック強化（第2弾）
- ホーム: タイトルロゴを大きく堂々と（clamp下限を上げる）、ボタンを大型化しゴールドの発光縁
- 馬選択カード: 馬ごとのアイキャッチ（GPT Image 2で馬の顔アイコン8種を生成 `assets/art/horses/horse1.png`〜`horse8.png`、円形フレームで表示。生成が難しければ絵文字を金枠メダリオン化）
- レース画面: HUD（順位表・ログ）をガラスモーフィズム化、レース開始時「スタート！」演出、ゴール時「FINISH」フラッシュ
- 結果画面: 1〜3着の表彰台風レイアウト（金銀銅）、払い戻しの数字カウントアップ
- 3D: ゴール板・電光掲示板の質感向上、トラック内側に花壇や池などの装飾、夕方バリエーション（レース毎ランダル）

## 制約（前回と同じ・必読）

- ビルドなし静的サイト維持、Vanilla JS + three.js (importmap)
- レスポンシブ必須（375px縦〜32インチワイドで背景が必ず全面を覆うこと）
- 動的カメラ「全馬が常に画面内」のロジックに手を入れない
- 2Dフォールバック（src/race.js）を壊さない
- 完了時 `src/version.js` を APP_VERSION 0.5.0 / APP_BUILD 8 に、index.htmlのバッジも更新
- タスクごとに日本語メッセージで独立コミット。push・デプロイはしない
