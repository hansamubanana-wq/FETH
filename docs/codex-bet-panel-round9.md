# Codex実装指示書: ベットパネルを右サイドへ移動（v0.12.0）

## ユーザーからの指摘

馬を選んだあとに出る「賭け式選択・金額選択」パネルが**画面下部に出てくるのが使いにくい**。
指定は「画面中央」または「馬を左に寄せて右側にパネル」のどちらか。
→ **右サイドパネル方式を採用**する（理由: 馬単・3連単など賭け式選択後にさらに馬を選ぶ必要があり、
中央モーダルだと馬一覧が隠れて操作できなくなるため。横画面標準なので横幅も活かせる）。

## 実装内容

### レイアウト（横画面・PC）
- 馬選択画面を**2カラム**にする: 左=馬カードグリッド、右=ベット操作パネル
- 右パネルは `position: sticky` などで**スクロールしても常に見える**ようにする
- 幅の目安: 右パネル 300〜360px、残りを馬グリッドに。馬グリッドは3〜4列を維持
- 右パネルの中身は現行の内容をそのまま移植:
  - 選択中の内容（例「馬単: 1.アカツキロウ → 2着になる馬を選択中」）
  - 賭け式ボタン（馬を選ぶと表示）
  - 金額入力（100/500/1000/半分/全部＋手動）※賭け式と馬が確定してから表示
  - 「この内容で賭ける」ボタン（未完了時は理由を表示してdisabled）
  - 購入済み馬券リスト（ベットスリップ）と「ベットを終了する」
- **馬を選ぶ前**は右パネルに「馬を選んでください」等のプレースホルダを出し、レイアウトが動かないようにする
  （パネルが出たり消えたりして馬グリッドの幅が変わるのは避ける）

### 縦画面・狭い画面（フォールバック）
- 幅が足りない場合（例 900px未満）は2カラムをやめ、現行どおり下部に積む1カラムに戻す
- ただし現行の「下に出てくる」挙動でも、**選択中の内容と確定ボタンは画面下部に固定表示**して見失わないようにする

### 動作は変えない
- 「馬を選ぶ→賭け式→（必要なら追加の馬）→金額→確定」の一本道フロー自体は現行のまま
- 確定後に初期状態へ戻る挙動も維持
- engine.js の判定ロジックには触らない

## 制約（毎回同じ・必読）

- ビルドなし静的サイト維持、Vanilla JS + three.js (importmap)
- レスポンシブ必須（375px縦〜ワイド、および横画面 844x390 で崩れないこと）
- 完了時 `src/version.js` を APP_VERSION 0.12.0 / APP_BUILD 16 に、index.htmlのバッジも更新
- タスクごとに日本語メッセージで独立コミット。push・デプロイはしない
