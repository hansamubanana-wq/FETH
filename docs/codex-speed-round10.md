# Codex実装指示書: レースの疾走感を出す（v0.13.0）

## ユーザーからの指摘

「レース場が小さいのか、馬の疾走感がない。会場大きくして速度上げるといいかも。」

## 分析（実装前に必ず読むこと）

動的カメラが馬群を追従するため、**馬は画面内でほぼ同じ位置に留まり続ける**。
そのため速度の手がかりは「地面・背景が後方へ流れる速さ（オプティカルフロー）」しかない。
現状 `MIN_VIEW_HEIGHT = 34` / `FULL_VIEW_HEIGHT = 64`（race3d.js）で、
バランス調整後は馬群が密集したため常時ズームイン寄りになっている。

つまり「速く感じない」の主因は馬の絶対速度そのものではなく、
**画面内を流れる視覚的リファレンスの乏しさ**である可能性が高い。単に速度定数を上げるだけでは
「早送り」に見えて不自然になるので、以下を組み合わせること。

## 実装内容

### 1. 会場を大きく＆スケール感を出す
- コースの物理スケールを拡大し、それに合わせてカメラ距離・視野も調整
- コース脇の視覚的リファレンスを増やす: ハロン棒（距離標）、ラチの支柱、
  芝の刈り込みストライプの周期を細かく → **一定間隔で流れていく物体**が速度感を生む
- スタンド・照明塔・樹木など、遠景と近景の視差（パララックス）が出る配置に

### 2. 体感速度を上げる
- 馬の走行速度（＝レース所要時間）を上げる。ただしレースが短くなりすぎないよう
  距離とのバランスを取ること（目安: 現行より15〜30%速く、レース全体は現行の8割程度の時間）
- 脚のアニメーション再生速度を速度に連動させる（現状 `mixer.update(dt * 1.9)` 固定を見直す）

### 3. 演出で疾走感を補強
- 最終直線でカメラをやや低く・馬に寄せる（迫力重視のカット）。
  ただし**全馬が画面内に収まる制約は絶対に守る**こと
- 速度に応じた軽いモーションブラー相当の表現（フルスクリーンのポストエフェクトは重いので、
  スピードライン用のパーティクルや、画面端のわずかなビネット揺れなど軽量な手法で）
- 砂埃・芝の飛沫を速度連動で増やす（既存のパーティクルプールを活用）
- 風切り音のような演出は不要（音は今回対象外）

## 制約（毎回同じ・必読）

- ビルドなし静的サイト維持、Vanilla JS + three.js (importmap)
- **動的カメラの「全馬が常に画面内」ロジックは壊さないこと**。
  スケール変更やカメラ変更のあと、必ずレースを流して全馬がフレームアウトしないことを確認する
- 2Dフォールバック（src/race.js）も同じ体感速度になるよう追随させる
- オンライン対戦のシード同期（同一シード→同一結果）を壊さない
- 馬券判定・払い戻しロジックとレースバランス（着差・勝率分布）は変えない。
  速度を上げても**着順の決まり方は現状維持**であること
- パフォーマンス: レース中60fps（PC）/30fps以上（モバイル）
- 完了時 `src/version.js` を APP_VERSION 0.13.0 / APP_BUILD 17 に、index.htmlのバッジも更新
- タスクごとに日本語メッセージで独立コミット。push・デプロイはしない
