# Codex実装指示書: グラフィック強化 第3弾（v0.6.0）

前回までの続き。GPT Image 2（image 2.0）を積極的に使い、さらにグラフィックの完成度を上げる。

## タスク

### 1. レース中の3D演出強化 (src/race3d.js)
- 馬体のカラーバリエーション: 馬選択カードの肖像アート8種（assets/art/horses/）と毛色の対応を取り、3Dの馬の色味を8頭で差別化（GLB・フォールバック両方）
- 走行時の砂埃・芝の飛沫パーティクル（先頭集団の足元、モバイルは数を半減）
- 最終直線で観客席のフラッシュ（カメラのフラッシュ風の点滅ライト）
- ゴール後のウィニングラン中、1着馬にスポットライト

### 2. 勝利演出のキーアート (結果画面)
- GPT Image 2で `assets/art/victory.png`（優勝の祝祭キーアート、金の紙吹雪と光、中央は空けて馬肖像を重ねられる構図）を生成
- 結果画面の表彰台背後に飾り、1着の馬肖像を大きく中央に合成表示

### 3. 細部の磨き込み (UI)
- 賭け式ボタン（単勝/複勝/…）にアイコン的装飾と選択中のゴールドアンダーライン
- 画面遷移を0.25sのフェード＋わずかなスケールでリッチに
- 「賭け方のコツ」ガイド画面にも背景とパネル装飾を適用（現状素っ気なければ）
- コイン表示に金貨アイコン（CSS/SVGでよい）と増減時のバウンス

### 4. ローディング体験
- 初回ロード時のスプラッシュ（ロゴ＋ゴールドのスピナー、アセット読込完了でフェードアウト）
- 3Dレース準備中のプログレス表示

## 制約（毎回同じ・必読）

- ビルドなし静的サイト維持、Vanilla JS + three.js (importmap)
- レスポンシブ必須（375px〜ワイド）
- 動的カメラ「全馬が常に画面内」のロジックに手を入れない
- 2Dフォールバック（src/race.js）を壊さない
- パフォーマンス: レース中60fps（PC）/30fps以上（モバイル）。パーティクルはプール化
- 完了時 `src/version.js` を APP_VERSION 0.6.0 / APP_BUILD 9 に、index.htmlのバッジも更新
- タスクごとに日本語メッセージで独立コミット。push・デプロイはしない
