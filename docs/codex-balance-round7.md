# Codex実装指示書: レースバランス調整（v0.10.0）

## 問題（ユーザー報告）

- 1位と最下位の着差が大きすぎる
- オッズ上位の馬が不自然に遅くなり、下位馬が勝つケースが多すぎる
- 「実力差を覆した番狂わせ」ではなく「ほぼ運」に感じられる

## 目指すバランス（ユーザー指定・そのまま設計目標とする）

- 上位人気馬は基本的に強く、安定して上位争いをする
- 下位人気馬の勝利は珍しいが、条件が噛み合えば起こり得る
- 番狂わせが起きても「上位馬が不自然に失速」ではなく「下位馬が好走した結果」として表現する
- ランダム性は順位を完全に決めるのではなく、実力差の範囲内で結果を揺らす要素にする
- 1位と最下位の着差は、特殊な事情（スタミナ切れ・重大な展開不利・適性不足など）がない限り現実的な範囲に収める

## 実装方針

対象は `src/engine.js`（レース生成・能力）と `src/race.js`（事前計算シミュレーション）。

1. **ベース速度の分散を縮小**: 能力値→速度への変換カーブを圧縮し、最上位と最下位の素の速度差を小さくする（着差の目安: 通常レースでトップと最下位が数馬身〜十数馬身以内。ゴール映像で画面内に全馬が収まる程度）
2. **ノイズの構造を変更**: 現在の「大きな一様ランダム」をやめ、
   - 毎レースの調子（小幅・正規分布的）
   - 展開・位置取り（レース中の局面ボーナス/ペナルティ、小幅）
   - スタミナ配分（終盤の伸び/失速。脚質と対応）
   の合成にする。**ノイズの合計振れ幅は実力差より一段小さく**し、実力が近い馬同士でのみ順位が入れ替わりやすくする
3. **番狂わせの表現**: 下位馬が勝つときは「調子・展開・適性が全部噛み合って僅差で差し切る」形にする。
   上位馬に大きなマイナスを引かせて実現しない（上位馬の失速要因は明確な理由があるときだけ）
4. **大差の限定**: スタミナ切れ・展開不利・適性不足が重なった馬のみ大きく離される。それ以外は集団内に収める
5. **オッズとの整合**: 調整後の実際の勝率とオッズが大きく乖離しないよう、オッズ計算も同じモデルで再計算する
6. **検証**: モンテカルロ（数千レース）をNodeで回し、以下を確認してコミットメッセージか docs に数値を残す:
   - 1番人気の勝率がおおむね25〜40%、複勝圏率が高い
   - 最低人気の勝率が数%以下だがゼロではない
   - 平均着差・最大着差が縮小していること

## 制約

- 馬券の的中判定・払い戻しロジックは変えない
- 3D/2D描画やカメラには手を入れない（着差が縮むことで映像は自然に密集する）
- 特殊能力（怪物・まくり等）の演出は残すが、効果量は新バランスに合わせて再調整
- オンライン対戦のシード同期（同一シード→同一結果）を壊さない
- 完了時 `src/version.js` を APP_VERSION 0.10.0 / APP_BUILD 13 に、index.htmlのバッジも更新
- タスクごとに日本語メッセージで独立コミット。push・デプロイはしない
