# Codex実装指示書: グラフィック強化 第4弾（v0.7.0）

GPT Image 2（image 2.0）を積極活用してさらに磨く。今回は「まだ手つかずの画面・状態」と「レースの臨場感」。

## タスク

### 1. 天候・時間帯システム (src/race3d.js)
- レース毎ランダム（シード決定論的）で「昼晴れ／夕方／夜（ナイター照明）」の3バリエーション
- 夜: 照明塔のライト・馬場を照らす光だまり・空に星。既存の夕景と合わせて計3種
- 雨天（任意・余力があれば）: パーティクルの雨＋馬場の色を湿った色に

### 2. レースの臨場感
- コース内側の大型ビジョン（電光掲示板）に現在の順位トップ3をリアルタイム表示（CanvasTextureを毎秒更新、負荷に注意）
- スタートゲートの造形改善と、スタート時にゲートが開く演出
- 3着までゴールした瞬間の着順テロップ（3Dオーバーレイでなく HUD でよい）

### 3. オンライン系画面のグラフィック (index.html / style.css)
- ルーム作成・合言葉入力・待機画面に、ベット画面と同等のパネル装飾・背景適用を徹底
- 待機中プレイヤーリストをカード化（金縁アバター枠、GPT Image 2で汎用アバター背景1枚生成可）

### 4. 2Dフォールバックの見栄え (src/race.js)
- WebGL不可端末用の2D描画を、現行の簡素な表現から「横スクロールのレトロ競馬」風に強化
  （空・観客席・芝のパララックス背景、馬は肖像アート流用の走行アニメ風）
- 過度に作り込まない。3Dの代役として恥ずかしくないレベルでよい

## 制約（毎回同じ・必読）

- ビルドなし静的サイト維持、Vanilla JS + three.js (importmap)
- レスポンシブ必須（375px〜ワイド）
- 動的カメラ「全馬が常に画面内」のロジックに手を入れない
- パフォーマンス: レース中60fps（PC）/30fps以上（モバイル）。CanvasTexture更新は1秒間隔まで
- 完了時 `src/version.js` を APP_VERSION 0.7.0 / APP_BUILD 10 に、index.htmlのバッジも更新
- タスクごとに日本語メッセージで独立コミット。push・デプロイはしない
