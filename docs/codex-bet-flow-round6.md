# Codex実装指示書: ベットフロー刷新（v0.9.0）

前回導入した「①②③ステップ表示」の仕組みは**廃止**する。
賭け金と賭け式が別々の場所にあって見づらい、という指摘への根本対応として、
**馬を起点にした一本道のフロー**に作り直す。

## 新しいフロー（ユーザー指定）

1. **初期状態は馬カードだけ**を見せる（賭け式ボタン列・金額入力は最初は出さない）
2. **馬を1頭タップしたら**、その場で「賭け式を選択」パネルが出てくる
   （選んだ馬の近く、またはモーダル/ボトムシートで。単勝/複勝/馬連/馬単/ワイド/3連複/3連単）
3. 賭け式を選んだら、**その賭け式に必要な残りの選択を促す**
   - 例: 馬単なら「2着になる馬を選んでください」、3連複なら「あと2頭選んでください」
   - 順序が意味を持つ式（馬単・3連単）は「1着→2着→3着」の順で聞き、選択済み馬に順序バッジ
   - 途中で選び直し・キャンセルができること
4. 必要な馬を選び終わったら、**最後に賭け金の入力/選択**（プリセット 100/500/1000/半分/全部 ＋ 手動入力）
   と想定払戻を表示し、「この内容で賭ける」で確定
5. 確定後は初期状態（馬カードだけ）に戻る。購入済み馬券は下部のベットスリップに蓄積（取消可）

## 実装上の注意

- 1画面に常時全部出すのではなく、**今やるべき操作だけが見えている**状態を作るのが目的
- 進行中の選択状態は画面上部に小さくパンくず表示（例: 「馬単: 1着[3] → 2着を選択中」）
- 「ベットを終了する」ボタンは従来どおり（全プレイヤーのベット完了でレースへ）
- オンラインモードのベット画面too同じフローに統一
- 既存の engine.js の馬券判定ロジックには手を入れない（UIの組み替えのみ）
- レスポンシブ必須。スマホでは賭け式選択・金額入力をボトムシート的に出すと親指で操作しやすい

## 制約（毎回同じ・必読）

- ビルドなし静的サイト維持、Vanilla JS + three.js (importmap)
- 完了時 `src/version.js` を APP_VERSION 0.9.0 / APP_BUILD 12 に、index.htmlのバッジも更新
- タスクごとに日本語メッセージで独立コミット。push・デプロイはしない
