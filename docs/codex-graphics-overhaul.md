# Codex実装指示書: グラフィック全面刷新（v0.4.0）

## ゴール

ホーム画面・UI・3Dレース描画の全てを「格段に」リッチ化する。
テーマは現行を踏襲: **ダーク×ゴールドの高級競馬場**（style.css の `--panel` / ゴールド基調）。

画像アセットは **GPT Image 2（image 2.0）で自分で生成**して `assets/art/` に配置すること。
生成できない環境の場合のみ、CSS/three.jsのプロシージャル表現で代替してよい。
必要アセット: home-hero.png（16:9 ホーム背景・ダーク×ゴールドの夜の競馬場）、
logo.png（エンブレム、正方形）、ogp.png（1200x630）、favicon.svg、PWAアイコン192/512。

## 制約（必読）

- ビルド工程なしの静的サイトを維持。Vanilla JS (ES Modules) + three.js (importmap/unpkg) のまま
- 依存追加はCDN経由のみ。バンドラ導入禁止
- スマホ〜PCのレスポンシブを崩さない
- レース中の動的カメラは「全馬が常に画面内」が仕様。カメラ・描画変更後は必ず馬がフレームアウトしないこと
- 低スペック端末向けにWebGL失敗時の2Dフォールバック（src/race.js）を壊さない
- 完了時に `src/version.js` の APP_BUILD を+1、APP_VERSION を 0.4.0 に。index.html のバッジも更新

## タスク分解

### 1. ホーム画面 (index.html + style.css)
- `assets/art/home-hero.png` をヒーロー背景に（暗めのオーバーレイ＋ゴールドのビネット）
- `assets/art/logo.png`（タイトルエンブレム）をタイトル上部に配置。テキストタイトルは残す（アクセシビリティ）
- ボタンをゴールドグラデ＋ホバー時のシャイン（::after で光が走る）に
- 背景に微細なパーティクル（CSSまたは軽量canvas、20個以下）

### 2. 画面UI全般 (style.css / src/betui.js / src/raceui.js)
- パネルに金縁の装飾ボーダー（border-image または擬似要素）
- 馬選択カード: 馬毎のカラーリボン、能力メーターをグラデ化、選択時のゴールドパルス
- 結果画面: 1着馬の演出強化（トロフィー、光芒、着順の順次スライドイン）
- オッズ・コイン表示に数字ロール（カウントアップ）アニメーション

### 3. 3Dレース (src/race3d.js)
- 地面・コース: 芝/ダートに CanvasTexture ベースのプロシージャルテクスチャ（縞刈り芝、ラチの白柵の質感向上）
- 空: グラデーションSky + 雲（billboard sprite）。時間帯バリエーション（昼/夕方をレース毎にランダム）は任意
- 観客席: 現状より密度を上げ、色ノイズでにぎやかに。旗の揺れアニメーション
- 馬: GLBが読める場合の質感向上（ライティング調整）、フォールバック馬のシルエット改善
- ライティング: 影付きDirectionalLight + 半球光。モバイルでは影OFFの品質分岐
- ゴール演出: 紙吹雪の量・色を強化、スローモーション着差リプレイは任意（大きければ次版へ）

### 4. メタ・アイコン（開発ルール準拠）
- `assets/art/ogp.png` を og:image / twitter:card に設定
- favicon.svg（エンブレム簡略版）、PWA用 192/512 アイコン
- README.md のスクリーンショット更新

## 進め方

- 1→2→3→4 の順で、各タスクを独立コミット（日本語コミットメッセージ）
- 各段階でローカル対戦を1周（ホーム→馬選択→ベット→レース→結果）して確認
- パフォーマンス目標: レース中60fps（PC）/ 30fps以上（モバイル）。draw call増に注意
